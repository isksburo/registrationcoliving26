# 🏥 ПАЛАТА №6 — форма регистрации

Нарочно кривая форма регистрации для вечеринки-фандрайзера «Палата №6» (коливинг #6).
Гостю — ад и тормоза, организатору — чистые данные. Подробности: [`docs/SPEC.md`](docs/SPEC.md).

## Запуск на Mac Mini
Двойной клик по **`start.command`** (или в терминале `./start.command`).
Поднимется локальный сервер на `127.0.0.1:6006` и откроется Chrome в режиме киоска.
Остановить — закрыть Chrome и `Ctrl+C` в терминале.

> Нужен `python3` (есть в macOS после `xcode-select --install`) и Google Chrome.

## Страницы
| Файл | Что | Кому |
|------|-----|------|
| `index.html` | Выбор скина + точка сброса между гостями | организатор |
| `registratura.html` | Скин А — «Советская регистратура» | гость |
| `inyerface.html` | Скин Б — «Анти-UX арт» | гость |
| `admin.html` | Ординаторская: таблица, «оплачено», CSV | главбух |

## Админка
Тайный вход: **6 кликов** по заголовку на стартовом экране (или напрямую `…/admin.html`).
Там — все записи, тумблер «оплачено» (ставишь, когда конверт реально у главбуха) и кнопка **⬇ CSV**.

## Данные
Лежат в `localStorage` браузера на этой машине (ключ `palata6_registrations`).
Никуда не уходят, пока сам не выгрузишь CSV. **Перед уборкой квартиры — скачай CSV.**

## Правка
- `shared.js` — хранилище, театр зависаний, киоск. Менять глубину «ада» (тайминги зависаний, лаг ввода) — здесь.
- Скины — внешний вид и набор шагов. Логику сохранения не трогать: сначала `Palata.Store.create/update`, потом глюки.
